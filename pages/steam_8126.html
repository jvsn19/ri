Early Access Software

Get involved with this software as it develops.

Note: This Early Access software is not complete and may or may not change further. If you are not excited to use this software in its current state, then you should wait to see if it progresses further in development. Learn more

What the developers have to say:

Why Early Access?

“Early Access will allows us to ensure stability and performance tuning for a wider range of systems and focus on developing features that users want the most.”

Approximately how long will this software be in Early Access?

“We are hoping to keep the program in EA as short as possible. Currently we are looking for 3-6 months of Early-Access period to develop new features, APIs and the AI-Assistant part and make sure the Mascot Characters does not have bugs, technical problems or performance issues on Early Access users.”

How is the full version planned to differ from the Early Access version?

“The main major and eye-catching feature for which we are opting for early access is the development of AI-Assistant and integration of a variety of utility APIs and Cloud Services. By the time we leave Early-Access, we are hoping the AI-Assistant would be production and real-world ready. Other than that, there will be a large number of 3D/2D Mascot Characters available to choose from to have them live on your desktop.
Last but not least, we hope to have a full fledged Workshop Integration to allow automated integration of user submitted models, services, APIs and content to the project.”

What is the current state of the Early Access version?

“Desktop Mascot Engine currently has multiple fully working Mascot Characters and only lacks the AI-Assistant part of the project. You can chose a Mascot Character to have it live on your desktop and fiddle/interact with it for it to respond back to your interactions.”

Will this software be priced differently during and after Early Access?

“The Project will gradually rise in price as new major features are added to it during and after Early Access. We are planning to have certain inAppPurchases and crowdfunding/Patreon goals to support its MachineLearning development costs.”

How are you planning on involving the Community in your development process?

“We wish to collect feedback and suggestions to create more features and contents, and are hoping the community itself will speed up the development of the project as a Community Contributed Project.
We will also be creating APIs and Guides to allows users to write AI/Assistant code and services themselves and submit them for integration to main project.”

19 June

DevLog #4 - Settings, New Features and Upcoming stuff



Okay, so first of all, sorry for almost a month delay for this update. Initially this update was supposed to be released 2 weeks back but unexpected delays and some real-life problems caused this delay.
However, this delay gave us more time to try our hands on some more experimental features, fix some bugs on them and bringing one of our most crucial features near release - Animation Motion (see below).

Lets get straight to what the new v6.19.2018 update includes

Settings
----------------------
Settings are finally available. Most of the settings are self-explanatory with tooltip messages to show what each setting does, but feel free to ask in community discussions if you are confused about any setting or want any particular setting added.
There are a few settings missing right now which we will add by this weekend, including:
-Last Used Mascot
-Windows Startup
-Auto Hide Mascot (when playing a steam game)
-Reset Mascot Position (specially useful when Motion Animation is released)
-Reset Mascot Form/Area (if for whatever reason you take it out of screen)
-Single Animation Mode
-Motion Animation
-Change Language (Interface)
-Individual Audio Volumes (AI Voice, Mascot Expressions and Audio Effects/others)

The Show Mascot Outline option in debug section is really good to get a visual idea of how big your mascot play area is (as it affects performance and cpu usage).



System Tray Icon
--------------------------------
Finally, we were able to make this work without breaking the Mascot's rendering engine. You can now Minimize the main interface like a normal window or Close/Hide it to the system tray.
Just to clear any confusion, the Exit/Quit ("X") button on the interface menu will close the interface and hide it (and the taskbar icon/button) to the System Tray. You can open the interface again by double clicking the tray icon or right clicking it and using "show interface" option.
To actually quit the Desktop Mascot Engine, you need to use the "Quit" option/button from the System tray Icon (right click on it).




New Characters
--------------------------------
4 new characters added again. Decided to add a new Mecha mascot character again due to popular demand.
We have also added some new Dancing animations to the human mascot characters (which use our new Animation system).






New Audio Responses
--------------------------------
We have added some more and new Audio Responses to the Anime Charas for more Animation States (happy, sad, tired, playful). We will be adding Voice Expressions for some more animation states in coming days.
We will also be adding some experimental Voice Expressions to the Mecha/Robot and Zombie charas.
Further, we have added some music effects for particular animations (guitar playing, gangnam style dance, chicken dance).

Localization
--------------------------------
Some localization improvements were done and localization was added for settings and tray icon texts.
Please report and help us improve the translations in community discussions if you find something wrong.


Upcoming stuff, what we are working on
--------------------------------
1. Workshop
=> Workshop has been our priority since start but we want to make sure we implement this right and not break anything. Workshop is taking time as we need to add sanity checks for anything and everything the users will submit/upload and to add proper curated content checks to transition the workshop easily to open content model.

2. Motion Animations
=> Almost done. We hope to release this feature in a day or two and this will spice up your mascots more as they will be able to animate and move around your desktop rather than animating at same spot.

Static Animations animate at a single position


Motion Animation will allow the character to move around as per its animation's motion



3. AI Assistant
=> We have slowly started working on the AI Assistant side by side. We expect to keep releasing it in small parts along with our next updates.
Expect a working Conversational AI Assistant in coming days. For starters, you will be able to converse with the AI and get a feel of how it would work and respond once we start adding services and functionalities to it.
For now, we are adding some casual conversations, question-answers, jokes, etc. that you can ask the AI.

4. Performance
=> This has been reported a few times by Intel CPU users about the Odd high cpu usage on low-end (G4560) and high-end (i7-7700) cpus alike. So far, we have tracked down the problem to be specific to direct-X related and are working on finding a fix for it (tbh, its quite related to how DX and Unity work with each other). We will be exploring openGL to see if it can improve the usage.

5. 2D Characters
=> Since many people have been asking for it, we will start working on adding our very first 2D character soon. We will also provide guides and tutorials to customize and submit your own charas.
Moving forward, we will now be focusing more on features and small utilities rather than characters.

About This Software

Desktop Mascot Engine allows you to have a character mascot live on your desktop and interact with it in real-time and have it respond to your interactions.

If you remember those old Stick-Fight videos on youtube, where the stick-man's were fighting on Windows Desktop, thats pretty much how the character/mascot looks on your desktop. Think of it as a living Game/Anime/3D-Model Character that overlaps over Windows and other apps.

For now we are adding diverse variety character models as mascots with some animations to allow you to relax/fiddle with the mascots to distract yourself from stress and stuff.

====================
This is the early access build.
Interaction/Control is only possible directly when clicking (left mouse button) on the character/mascot.

1. To change the characters, press and hold mouse button 1 (left click) on the character and press left or right key to toggle through the initially available 4 characters.
2. To change the animation state of character, just left mouse click on the character itself.
Animation change is random and sometimes its possible same animation plays, if so, left click again to change to different animation.
3. To change the mascot size, left click on mascot and use mouse scroll wheel to increase or decrease size.

We are looking for more suggestions and input related and unrelated to the AI/Assistant part, what you would like the mascot/character to do, what features you would expect and what particular characters/genres you would like ?
